A couple of weeks ago I was working on a very tricky issue on ProxyManager.

The problem is simple: instantiating ReflectionClass or ReflectionProperty is slow, and by slow, I mean really slow!

The reason for this reasearch is that I'm trying to optimize a "hydrator" to work with larger data-sets by still keeping a low initialization overhead.

PHP 5.4 to the rescue!

PHP 5.4 comes with a new API for Closures, which is Closure#bind().

Closure#bind() basically allows you to get an instance of a closure with the scope of a given object or class. Neat! That's basically like adding APIs to existing objects!

Changing closure scope vs. Reflection: performance

How does this technique compare with ReflectionProperty#getValue()? Is it actually faster?

I've built a simple benchmark to profile the "setup" step for this trick over a million iterations:

{% highlight php %}yummy; }, null, 'Kitchen'); }{% endhighlight %} {% highlight php %}setAccessible(true); }{% endhighlight %}

Turns out that on my pc (PHP 5.4.9 ubuntu 13.04 box), the first script takes ~ 1.3 seconds to run, while the second one requires ~ 2.6 seconds. Reflection is slower here.

That's completely un-interesting, since nobody will ever instantiate a million reflection properties, or at least I cannot find a good reason to do that.

What seems to be more interesting is how accessing properties compares. I've profiled that too:

{% highlight php %}yummy; }, null, 'Kitchen'); for ($i = 0; $i < 1000000; $i += 1) { $sweetsThief($kitchen); }{% endhighlight %} {% highlight php %}setAccessible(true); for ($i = 0; $i < 1000000; $i += 1) { $sweetsThief->getValue($kitchen); }{% endhighlight %}

I would actually have expected the reflection system being very slow in this case, but in fact it is much faster!

The first script took ~ 3.4 seconds to run, while the second one just needed ~ 1.4 seconds!

That basically tells us that the instantiation time of reflection classes is negligible, and that it's actually faster than userland code in accessing properties.

Accessing private class properties by reference

There's actually one big advantage in using a Closure instead of ReflectionProperty, which is that you can now retrieve a private property by reference!

{% highlight php %}yummy; }, null, $kitchen); $cake = & $sweetsThief($kitchen); $cake = 'lie'; var_dump('the cake is a ' . $sweetsThief($kitchen));{% endhighlight %}
